# 主机管理系统（Django+DRF）开发思路
## 一、项目核心定位
基于 Django + Django REST Framework（DRF）开发的轻量级主机管理系统，核心实现主机信息的增删改查、机房/城市关联管理、定时任务自动化运维（密码重置、数据统计），并通过中间件实现请求耗时监控，适配前后端分离的接口开发模式。

## 二、核心技术栈与分层设计
### 1. 核心技术栈
- 后端框架：Django（ORM 数据建模）+ DRF（RESTful API 开发）；
- 异步/定时任务：Celery（实现密码批量重置、每日主机统计）；
- 辅助工具：Git（版本控制）、中间件（请求耗时监控）。

### 2. 分层设计
| 层级       | 作用                                                                 |
|------------|----------------------------------------------------------------------|
| 模型层（Model） | 定义 City（城市）、IDC（机房）、Host（主机）、HostStatistics（统计）模型，通过外键实现城市-机房-主机的关联关系，封装密码加密、随机生成等核心方法； |
| 序列化层   | 基于 DRF ModelSerializer 实现数据序列化，通过 source 展平关联字段（如 idc_name/city_name），配置 write_only 隐藏敏感字段（如 root_password）； |
| 视图层     | 基于 DRF ModelViewSet 实现 CURD 接口，重写 perform_create/perform_update 处理密码加密等业务逻辑； |
| 任务层     | 用 Celery @shared_task 定义异步/定时任务，实现每8小时重置主机密码、每日0点统计各机房主机数量； |
| 中间件层   | 自定义请求耗时中间件，统计请求全生命周期耗时，通过日志/响应头输出；       |

## 三、核心业务实现思路
### 1. 数据关联设计
- 城市（City）→ 机房（IDC）：一对多关联；
- 机房（IDC）→ 主机（Host）：一对多关联；
- 统计（HostStatistics）：关联城市、机房，每日存储对应主机数量，避免实时统计性能损耗。

### 2. 关键功能实现
- 主机密码安全：前端传入明文密码，后端通过 set_password 加密存储，序列化器配置 write_only 确保密码不返回前端；
- 关联字段展平：序列化器自定义 idc_name/city_name 字段，通过 source 从关联模型取值，减少前端二次请求；
- 定时任务调度：Celery Worker 执行任务，Beat 负责定时触发（每8小时改密码、每日0点统计）；
- 请求监控：中间件记录请求开始/结束时间，计算耗时并输出日志、添加响应头 X-Request-Duration。

## 四、项目亮点
1. 数据分层清晰：通过 ORM 封装数据逻辑，序列化器解耦前后端数据格式，符合 RESTful 设计规范；
2. 性能与安全：异步任务避免耗时操作阻塞主进程，密码加密存储、敏感字段隐藏保障数据安全；
3. 可监控可扩展：请求耗时中间件便于性能排查，Celery 任务模式支持后续扩展批量运维、主机检测等功能。
